# VK Voice & Clips Downloader

`VK Voice & Clips Downloader` — это браузерное расширение на Manifest V3 для Chromium-совместимых браузеров, которое добавляет кнопки скачивания к голосовым сообщениям и VK Clips, уже доступным пользователю в интерфейсе `vk.com`.

Проект намеренно сделан лёгким: без сборщика, без фреймворков, без серверной части и без прямой интеграции с приватным API. Вся логика работает локально в браузере и опирается только на данные, уже присутствующие в React-дереве страницы.

Расширение ориентировано на ручную установку в режиме разработчика и наглядную демонстрацию подхода к работе с динамическим DOM, MV3 и page-owned React internals.

## Зачем нужен этот проект

У голосовых сообщений VK обычно нет прямого `audio src` в обычном DOM, а у клипов в DOM часто присутствует только временный `blob:` URL внутри `<video><source>`. Реальные ссылки на медиа лежат глубже, в данных React-страницы, например:

- `message.attaches.voice.linkOgg`
- `message.attaches.voice.linkMp3`
- `audioTrack.url`
- `children.0.props.children.props.video.files.mp4_480`
- `children.0.props.children.props.video.files.hls`

Этот проект показывает, как можно аккуратно построить над этим ограничением надёжное MV3-расширение и при этом сохранить код поддерживаемым, безопасным и пригодным для публичного портфолио.

## Ключевые возможности

- Добавляет компактную кнопку скачивания рядом с каждым голосовым сообщением
- Добавляет кнопку скачивания поверх карточек VK Clips
- Поддерживает динамически загружаемые сообщения и клипы через `MutationObserver`
- Извлекает ссылки на аудио и видео из React Fiber / React props
- Использует несколько fallback-путей на случай изменений во внутренней структуре VK
- Загружает файлы через стандартный браузерный `chrome.downloads`
- Хранит состояние ВКЛ/ВЫКЛ локально через `chrome.storage.local`
- Работает полностью на стороне пользователя

## Как это работает технически

### Поток выполнения

1. Content script находит на странице узлы `.AttachVoice` и `[data-testid="clips-feed-item"]`.
2. Для голосовых сообщений он добавляет кнопку внутрь `.AttachVoice__player`, а для клипов — кнопку в правом верхнем углу карточки.
3. При клике по кнопке content script временно помечает нужный DOM-узел специальным токеном.
4. Service worker запускает извлечение данных в `MAIN` world страницы через `chrome.scripting.executeScript(...)`.
5. Код в контексте страницы обходит ближайшие React internals и проверяет несколько известных путей.
6. Лучший доступный URL возвращается обратно в расширение.
7. Service worker запускает скачивание через API браузера с понятным именем файла.

### Извлечение ссылок для голосовых сообщений

Extractor проверяет несколько известных путей в React-данных:

- `memoizedProps.message.attaches.voice.linkOgg`
- `memoizedProps.message.attaches.voice.linkMp3`
- `pendingProps.message.attaches.voice.linkOgg`
- `pendingProps.message.attaches.voice.linkMp3`
- `memoizedState.memoizedState.current.audioTrack.url`

Приоритет URL:

1. `linkOgg`
2. `audioTrack.url`
3. `linkMp3`

### Извлечение ссылок для VK Clips

Для клипов расширение **не** использует `blob:` URL из DOM-дерева `<video>`, потому что это временная ссылка браузерного плеера, а не стабильный адрес файла для скачивания. Вместо этого расширение читает объект `video` из React Fiber / React props карточки клипа и выбирает лучшую доступную ссылку в таком порядке:

1. `files.mp4_480`
2. `files.mp4_360`
3. `files.mp4_240`
4. `files.mp4_144`
5. `files.hls`
6. `files.dash_sep`

Если доступен `mp4`, расширение скачивает именно его. Если в React-данных доступны только `hls` или `dash_sep`, расширение делает best-effort скачивание этой ссылки как есть и явно помечает это поведение в коде как fallback без ремультиплексирования.

Важно: ссылки на клипы могут быть временными и зависеть от текущей сессии пользователя и авторизации в VK.

По возможности расширение формирует понятное имя файла в виде `vk-clip-<owner_id>_<id>-<quality>.mp4`. Если качество неизвестно или доступен только stream-fallback, имя строится по доступным метаданным и типу источника.

### Почему здесь используется React Fiber

Расширение **не** пытается обходить авторизацию и **не** делает скрытых запросов к приватным внутренним endpoint'ам. Оно работает только с теми данными, которые уже доступны пользователю в открытом интерфейсе VK после загрузки страницы.

Практически это означает, что страница уже содержит метаданные голосового сообщения и клипа внутри React-объектов, но обычный DOM не даёт пригодный `audio[src]`, а у клипов часто содержит только временный `blob:`. Поэтому расширение использует инспекцию React internals страницы, а не пытается опереться на несуществующий или временный источник в DOM.

### Зоны ответственности

- `src/content/content-script.js`
  Общий runtime-слой content scripts: shared utilities, settings, module bootstrap и синхронизация состояния расширения.

- `src/content/voice-module.js`
  Поиск голосовых сообщений, инъекция кнопки в `.AttachVoice__player`, формирование имени файла и наблюдение за динамическими сообщениями.

- `src/content/clips-module.js`
  Поиск карточек VK Clips, инъекция кнопки поверх карточки, формирование имени файла и наблюдение за динамической лентой.

- `src/background/service-worker.js`
  Инициализация настроек, безопасная обработка загрузок и выполнение извлечения данных для голосовых сообщений и клипов через `chrome.scripting`.

- `src/popup/`
  Небольшой UI для локального включения и отключения расширения.

Более подробное техническое описание вынесено в [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).

## Установка

### Загрузка как распакованного расширения

1. Склонируйте или скачайте репозиторий.
2. Откройте `browser://extensions` или `chrome://extensions`.
3. Включите **Режим разработчика**.
4. Нажмите **Загрузить распакованное расширение** / **Load unpacked**.
5. Выберите корень репозитория.
6. Откройте `vk.com` и обновите страницу.

### Поддерживаемые браузеры

- Google Chrome
- Chromium
- Microsoft Edge
- Яндекс.Браузер
- Другие Chromium-совместимые браузеры с поддержкой Manifest V3

## Использование

1. Откройте страницу VK с голосовыми сообщениями или страницу VK Clips, уже доступную в вашем интерфейсе.
2. Найдите кнопку скачивания рядом с голосовым сообщением или в правом верхнем углу карточки клипа.
3. Нажмите на кнопку.
4. Браузер начнёт загрузку файла с осмысленным именем, если нужные метаданные доступны на странице.

Состояние popup:

- `Включено`: расширение добавляет кнопки скачивания на страницах VK
- `Выключено`: существующие кнопки удаляются, новые не вставляются

## Ограничения

- Расширение зависит от текущей DOM-структуры VK и внутренних React-данных страницы.
- Если VK изменит имена классов или пути к данным, логику извлечения может потребоваться обновить.
- Расширение работает только с тем контентом, который уже доступен пользователю в веб-интерфейсе VK.
- Состав имени файла зависит от тех метаданных, которые VK реально держит в текущем состоянии страницы.
- Ссылки на VK Clips могут быть короткоживущими и зависеть от активной авторизации пользователя.
- Если у клипа доступны только `hls` или `dash_sep`, расширение скачивает stream-URL как fallback и не конвертирует его в MP4.

## Безопасность, приватность и корректное позиционирование

- Нет автоматизации логина
- Нет работы с учётными данными
- Нет прямого клиента приватного API
- Нет серверной обработки
- Нет аналитики и телеметрии
- Нет извлечения данных за пределами уже отрисованного интерфейса

Настройки расширения хранятся локально через `chrome.storage.local`. Дополнительно см. [PRIVACY.md](PRIVACY.md) и [DISCLAIMER.md](DISCLAIMER.md).
